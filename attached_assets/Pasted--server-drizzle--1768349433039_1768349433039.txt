مهم جدًا قبل أي شيء:
- لا تعدّل أي شيء في server/ أو drizzle/ أو قاعدة البيانات أو ملفات التشغيل.
- التعديلات واجهة فقط داخل client/ وتحديدًا client/src.
- لا تغيّر المنطق أو الـ API أو الـ routes. فقط UI/UX وتحسين Responsive.
- لا تضف مكتبات جديدة إلا للضرورة القصوى، ويفضل استخدام الموجود بالفعل.

الهدف:
أبغى تحديث تصميم الموقع ليصير عصري وحديث ومرتب، مع دعم جوال ممتاز (Responsive + RTL مضبوط). ركّز على: القوائم (Navbar)، الأزرار، الألوان، المسافات، الخطوط، وتجربة الضغط (hover/focus/active).

قرار الألوان (ثابت):
اعتمد Palette واحدة: أخضر + ذهبي + لافندر
- الأخضر = اللون الأساسي (Primary)
- الذهبي = Accent (Highlights/CTA/Badges)
- اللافندر = خلفيات خفيفة/تدرجات/sections لإحساس عصري وناعم
مع تباين قوي للنصوص ووضوح ممتاز في الوضعين الفاتح والداكن.

المطلوب تنفيذه على 3 مراحل (نفّذهم بالترتيب، وتأكد كل مرحلة شغالة قبل الانتقال للي بعدها):

========================
المرحلة 1 — هوية وألوان (Theme) [ابدأ بهذه المرحلة فقط الآن]
========================
1) طبّق نظام ألوان واضح باستخدام الأخضر/الذهبي/اللافندر على:
   - الخلفيات (Backgrounds/Sections)
   - العناوين والنصوص (Typography colors مع تباين ممتاز)
   - الروابط (Links) وحالات hover/focus
2) الأزرار:
   - Primary = أخضر
   - Secondary = لافندر/محايد مع حدود
   - Accent/CTA = ذهبي عند الحاجة (مثل زر مهم أو Badge)
   - تأكد من حالات hover/active/focus ring بشكل أنيق ومتناسق
3) حافظ على Dark/Light mode إن موجود ولا تخربه، واضبط الألوان لتظل مقروءة في الوضعين.
4) لا تغيّر محتوى الصفحات أو النصوص، فقط الشكل.

مخرجات المرحلة 1:
- اذكر الملفات التي عدلتها (مساراتها).
- اذكر باختصار "قبل/بعد" من ناحية شكل الواجهة.
- اذكر أين تم تعريف الألوان (CSS variables أو Tailwind أو ملفات Theme).

========================
المرحلة 2 — Navbar والقوائم (Desktop + Mobile)
========================
بعد ما تنتهي من المرحلة 1 بنجاح:
1) حسّن شريط التنقل:
   - RTL مضبوط (اللوجو يمين، الروابط مرتبة)
   - تمييز واضح للصفحة الحالية (active)
   - أزرار "تسجيل الدخول" و"إنشاء حساب" تكون واضحة ومتناسقة مع الهوية
2) دعم الجوال:
   - على الشاشات الصغيرة حوّل الروابط لقائمة Menu (Drawer أو Dropdown)
   - زر القائمة واضح وسهل الضغط (tap target مناسب)
3) لا تغيّر routes أو أسماء الصفحات.

مخرجات المرحلة 2:
- اذكر الملفات التي عدلتها.
- وضح كيف أقدر أضيف/أعدل روابط القائمة لاحقًا.

========================
المرحلة 3 — تحسين عام للواجهة (Polish + Responsive)
========================
بعد نجاح المرحلة 2:
1) توحيد تصميم البطاقات (Cards): ظلال خفيفة + زوايا + مسافات متناسقة + حدود أنيقة.
2) تحسين Typography للعربي:
   - أحجام عناوين واضحة
   - line-height مناسب
   - تباعد أسطر يريح العين
3) تحسين تجربة الضغط للأزرار والحقول:
   - hover/focus واضح
   - حالات disabled/loading إن وجدت
   - مساحة لمس مناسبة على الجوال (حد أدنى 44px)
4) راجع الصفحات الأساسية (Home / PlanTrip / Packages / Dashboard إن وجدت) وتأكد أنها Responsive بالكامل.
5) اختبر على المقاسات: 320px, 375px, 390px, 768px, 1024px وأصلح أي كسر.

مخرجات المرحلة 3:
- قائمة الملفات المعدلة.
- Checklist سريع لاختبار الجوال (ماذا أشيك بالضبط).

ابدأ الآن بالمرحلة 1 فقط. لا تنتقل للمرحلة 2 حتى أطلب منك ذلك.
